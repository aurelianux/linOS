# linOS

Opinionated homelab and smart-home stack running on host "Manny" (Dell Wyse 5070, Arch Linux).

## Structure

This repository assumes all Docker Compose stacks live under the `stacks/` directory:

- `stacks/infra/` — MQTT (Mosquitto), Node-RED, and Tailscale VPN
- `stacks/proxy/` — Caddy reverse proxy in front of all web UIs
- `stacks/dns/` — AdGuard Home as local DNS / ad filter
- `stacks/homeassistant/` — Home Assistant Core and configuration
- `stacks/zigbee2mqtt/` — Zigbee2MQTT and coordinator
- `stacks/applications/`
  - `plane/` — Plane project management
  - `service-index/` — HTML overview page of all services (reachable at `manny.lan`)
  - `dashboard/` — linBoard dashboard (Web + API, reachable at `dashboard.manny.lan`)

Application source code lives in `apps/`:

- `apps/dashboard-web/` — React 19 + Vite + Tailwind + @hakit/core + Zustand frontend
- `apps/dashboard-api/` — Express 5 + Zod + Pino backend (BFF for non-HA data)

Configuration and tooling:

- `config/services.json` — health-check definitions (auto-generated by `update_index.py`)
- `config/dashboard.json` — dashboard entity configuration (rooms, entities, quick toggles)
- `scripts/` — operational scripts (`smrestart`, `smstatus`, `update_index.py`)
- `shell/manny.zshrc` — host shell aliases and functions

Runtime data (databases, logs, Caddy ACME, Zigbee DB, etc.) is excluded via `.gitignore` and never committed.

## Prerequisites

- Linux host with Docker and Docker Compose
- SSH access to the host (e.g. `192.168.2.31`)
- Node.js 20+ and pnpm 10.25+ (for local dashboard development)

## Initial Setup

```bash
# Copy the example config and fill in your values:
cp .env.linos.example .env.linos
# Enter your Tailscale auth key, host IP, HA token, etc.
```

> **Note:** `.env.linos` contains secrets and is not versioned.
> Never commit `.env.linos` directly — use `.env.linos.example` as a template.

## Starting the Base Stack

From the repo root (`linOS`):

```bash
cd stacks/infra && docker compose up -d
cd stacks/proxy && docker compose up -d
cd stacks/dns && docker compose up -d
cd stacks/homeassistant && docker compose up -d
cd stacks/zigbee2mqtt && docker compose up -d
```

Or restart everything at once:

```bash
./scripts/smrestart
```

## Applications

Service Index (landing page):

```bash
cd stacks/applications/service-index && docker compose up -d
```

Dashboard (linBoard):

```bash
cd stacks/applications/dashboard && docker compose up --build -d
```

Plane:

```bash
cd stacks/applications/plane/plane-app && docker compose up -d
```

## Dashboard Development

```bash
cd apps
pnpm install        # install all workspace dependencies
pnpm dev            # start frontend :4000 + API :4001 concurrently
pnpm build          # production build (Vite + tsc)
pnpm typecheck      # tsc --noEmit
pnpm lint           # ESLint
pnpm format         # Prettier
```

See [apps/dashboard-web/README.md](apps/dashboard-web/README.md) for frontend details and [docs/dashboard/](docs/dashboard/) for full development documentation.

## Important Paths for Backups

- Home Assistant config: `stacks/homeassistant/config/`
- Node-RED data: `stacks/infra/nodered/`
- Mosquitto data: `stacks/infra/mosquitto/data/`
- AdGuard data: `stacks/dns/work/`
- Zigbee2MQTT data: `stacks/zigbee2mqtt/data/`
- Caddy certificates/data: `stacks/proxy/data/` (Docker volume `caddy_data`)

## Scripts and Shell Configuration

### scripts/

| Script | Description |
|---|---|
| `scripts/smrestart` | Restart all stacks via `docker compose up -d` (reads `config/services.json`). Logs to `logs/` |
| `scripts/smstatus` | Show current container status. Logs to `logs/` |
| `scripts/update_index.py` | Regenerate `services.json` for the service index |

Scripts can be run directly from the repo — the path to the repo root is resolved automatically:

```bash
~/linOS/scripts/smrestart
~/linOS/scripts/smstatus
```

### shell/manny.zshrc

`shell/manny.zshrc` contains all Manny-specific aliases and functions (Docker shortcuts, `smrestart`, `smstatus`, `updateindex`, etc.). Include it in `~/.zshrc`:

```zsh
[[ -f ~/linOS/shell/manny.zshrc ]] && source ~/linOS/shell/manny.zshrc
```

After a `git pull`, changes are available in the next shell session automatically.

## Troubleshooting

- View logs: in the respective stack folder `docker compose logs`
- Restart Caddy: `cd stacks/proxy && docker compose restart`
- Restart Home Assistant: `cd stacks/homeassistant && docker compose restart`
- Restart Dashboard: `cd stacks/applications/dashboard && docker compose restart`
- Container status: `./scripts/smstatus` (or `dps` alias)
